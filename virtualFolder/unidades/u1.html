Evidencias - Paradigma de Aplicaciones Orientadas a Servicios

Servicios ofrecidos en la web

  • Servicios web públicos: Ej. APIs de Google Maps, Twitter, OpenWeatherMap (ofrecen datos/funcionalidades a terceros).
  • Servicios web empresariales: Ej. Sistemas de gestión de ventas, recursos humanos o facturación (para uso interno o entre empresas).
  • Servicios web de infraestructura: Ej. Almacenamiento en la nube (AWS S3), servidores virtuales (Azure) (soporte técnico y operativo).
  • Servicios web de comunicación: Ej. Correo electrónico, mensajería instantánea, videoconferencias.

Características de las aplicaciones orientadas a servicios

  • Reutilización de componentes/servicios en diferentes proyectos.
  • Desacoplamiento entre la lógica del servicio y su consumo.
  • Interoperabilidad entre plataformas (ej: Java con .NET).
  • Escalabilidad: Se pueden agregar/modificar servicios sin afectar el sistema completo.
  • Modularidad: Cada servicio cumple una función específica y bien definida.

Aplicaciones web híbridas (Mashup)

Las aplicaciones web híbridas, también llamadas Mashup, son aplicaciones que combinan información o servicios de diferentes fuentes para crear una sola aplicación. Estas fuentes pueden ser otras páginas web, APIs o servicios en línea.

Ejemplo: Un mapa que muestra ubicaciones de restaurantes, integrando Google Maps API (para el mapa) y Yelp API (para información de restaurantes).

Características clave:
-Utilizan datos de diferentes sitios o servicios
-Se apoyan en APIs para obtener información
-Funcionan a través de internet
-Integran contenido como mapas, videos, datos o redes sociales
-Son dinámicas, la información se actualiza constantemente
-Permiten reutilizar servicios existentes

Evidencias - Arquitectura Orientada a Servicios (SOA)

¿Qué es SOA?

Arquitectura de software que organiza la funcionalidad de un sistema en módulos independientes llamados "servicios", los cuales se comunican entre sí a través de interfaces estándar, permitiendo flexibilidad, reutilización e interoperabilidad entre diferentes plataformas y tecnologías.

Principios de diseño

  • Reutilización de servicios
  • Desacoplamiento entre proveedor y consumidor
  • Interoperabilidad entre tecnologías
  • Autonomía de cada servicio
  • Alta cohesión en la lógica de cada servicio

Estándares asociados

  • XML: Lenguaje de marcado para estructurar y transportar datos de forma legible por máquina y humano.
  • SOAP: Protocolo para el intercambio de mensajes estructurados en entornos distribuidos.
  • WSDL: Lenguaje para describir la funcionalidad de servicios web y cómo acceder a ellos.
  • UDDI: Directorio para registrar y buscar servicios web disponibles.
  • REST: Estilo arquitectónico que utiliza métodos HTTP (GET, POST, PUT, DELETE) para interactuar con recursos, basado en principios simples y escalables.
